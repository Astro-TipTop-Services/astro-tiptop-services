[TipTop v1.4.0](https://github.com/astro-tiptop/TIPTOP/releases/tag/v1.4.0), [P3 v1.5.1](https://github.com/astro-tiptop/P3/releases/tag/v1.5.1) and [MASTSEL v1.4.3](https://github.com/astro-tiptop/MASTSEL/releases/tag/v1.4.3) are here! 🎉  

This release introduces precision management, allowing users to trade a small amount of numerical accuracy for faster computations and lower memory usage.

✨ What’s new

- Added precision control in the `COMPUTATION` section.
- Default precision remains double precision (`float64`, `omplex128`) to ensure the same accuracy as previous versions.

- Optional single precision mode can now be enabled (see the [parameters page](https://astro-tiptop-services.github.io/astro-tiptop-services/docs/orion/parameterfiles#COMPUTATION))

🚀 Benefits of single precision:

- 10–20% faster computations
- ~30% lower memory usage

Small differences in results may appear (typically 1–2% in SR), but this option is particularly useful for large simulations or memory-limited environments.

🛠️ Additional updates

- Fixed several bugs identified during recent testing.

Update now and enjoy the latest improvements! <br/>

Check it out on PyPI 👉 [**astro-tiptop**](https://pypi.org/project/astro-tiptop/), [**astro-p3**](https://pypi.org/project/astro-p3/1.5.1/), [**mastsel**](https://pypi.org/project/mastsel/)
